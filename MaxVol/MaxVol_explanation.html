1. 문제 설정 — "무엇을 최대화하는가?"

MTP(Moment Tensor Potential)는 $n$개의 파라미터를 가집니다. 훈련 집합에 수천 개의 구조가 있을 때, 모든 구조를 써서 학습하면 비용이 크지만 사실상 정보가 중복되는 구조가 많습니다.

MaxVol은 다음 질문에 답합니다:

핵심 질문
전체 훈련 후보 구조 집합에서, $n \times n$ 행렬식(det)이 최대가 되는 $n$개의 구조를 어떻게 효율적으로 찾는가?
변수 n의 이중 의미

이 문서에서 n은 동시에 두 가지를 뜻합니다:

  • MTP 파라미터 수 — feature vector φ의 차원 (코드: CoeffCount(), 예: n = 256)
  • MaxVol이 선택하는 구조 수 — basis matrix A의 행/열 수

이 둘이 항상 같아야 하는 이유: A의 역행렬 A⁻¹이 존재하려면 A가 정방(square) 행렬이어야 합니다. 따라서 선택 구조 수 = feature 차원 = MTP 파라미터 수 = n 으로 고정됩니다. 나머지 후보 구조들은 m×n 행렬 B (m ≥ 1, m ≠ n)로 따로 관리합니다.

기하학적으로 이해하면 더 명확합니다.

$n$차원으로 일반화하면: $n$개의 구조 각각이 $n$차원 feature vector $\boldsymbol{\varphi}_i \in \mathbb{R}^n$를 가지고, 이들을 행(row)으로 쌓은 $n \times n$ 행렬 $\mathbf{A}$의 절댓값 행렬식 $|\det(\mathbf{A})|$이 최대인 부분집합을 선택합니다.

2. Feature Vector φ (MTP 특성 벡터)

MTP의 에너지는 MTP 파라미터 $\boldsymbol{\theta} \in \mathbb{R}^n$에 대해 선형입니다:

$$E(\text{cfg}; \boldsymbol{\theta}) = \sum_{i=1}^{N_\text{atoms}} E_\text{site}(i;\boldsymbol{\theta}) = \boldsymbol{\theta}^\top \boldsymbol{\varphi}(\text{cfg})$$

여기서 feature vector $\boldsymbol{\varphi}(\text{cfg}) \in \mathbb{R}^n$는:

$$\boldsymbol{\varphi}(\text{cfg}) = \frac{\partial E(\text{cfg};\boldsymbol{\theta})}{\partial \boldsymbol{\theta}} = \sum_{i=1}^{N_\text{atoms}} \frac{\partial E_\text{site}(i;\boldsymbol{\theta})}{\partial \boldsymbol{\theta}}$$
가중치에 따른 feature vector 구성
가중치포함되는 행행 수코드 함수
nbh_weight ≠ 0 원자별 site energy gradient $\frac{\partial E_i}{\partial \boldsymbol{\theta}}$ $N_\text{atoms}$ CalcSiteEnergyGrad()
ene_weight ≠ 0 전체 에너지 gradient $\frac{\partial E}{\partial \boldsymbol{\theta}}$ 1 CalcEnergyGrad()
frc_weight ≠ 0 힘 gradient $\frac{\partial F_{i,\alpha}}{\partial \boldsymbol{\theta}}$ $3 N_\text{atoms}$ CalcEFSGrads()
str_weight ≠ 0 응력 gradient $\frac{\partial \sigma_{\alpha\beta}}{\partial \boldsymbol{\theta}}$ 9 CalcEFSGrads()

기본값: ene_weight=1, 나머지 0. 즉 기본적으로 에너지 방정식 1개의 gradient만 사용합니다.

코드에서 이 행 수 계산은 active_learning.cpp:50–53에 명시되어 있습니다: 

// PrepareMatrix() — feature matrix B의 행 수 계산
int max_rows_cnt =
    (MV_nbh_cmpnts_weight != 0.0) * cfg.size()       // nbh: N_atoms rows
  + (MV_ene_cmpnts_weight != 0.0) * 1               // ene: 1 row
  + (MV_frc_cmpnts_weight != 0.0) * 3*cfg.size()   // frc: 3N rows
  + (MV_str_cmpnts_weight != 0.0) * 9;             // str: 9 rows

3. 행렬 A와 Basis — 훈련 집합의 기저

3.1 행렬 정의

$n = $ MTP 파라미터 수라 하면, MaxVol은 두 행렬을 관리합니다:

행렬크기의미코드
$\mathbf{A}$ $n \times n$ 현재 선택된 $n$개 구조의 feature vector를 행으로 쌓은 basis matrix MaxVol::A
$\mathbf{A}^{-1}$ $n \times n$ $\mathbf{A}$의 역행렬 (효율적 grade 계산용) MaxVol::invA
$\mathbf{B}$ $m \times n$ 새로운 후보 구조들의 feature vector ($m$개) MaxVol::B

3.2 초기화

처음에는 선택된 구조가 없으므로 $\mathbf{A}$를 작은 수 $\varepsilon$으로 초기화합니다:

$$\mathbf{A}^{(0)} = \varepsilon \cdot \mathbf{I}_n, \quad \varepsilon = 10^{-5} \text{ (기본값)}$$ 
// maxvol.cpp:29–36  — 초기화
for (int i = 0; i < n; i++)
    A(i, i) = init_treshold;       // A = ε·I

for (int i = 0; i < n; i++)
    invA(i, i) = 1.0/init_treshold; // A⁻¹ = (1/ε)·I

이는 "아무 구조도 선택되지 않은 상태"를 나타내며, 어떤 새 구조를 추가하든 grade가 매우 크게 나와 초기 선택이 빠르게 이루어집니다.

A, B 행렬의 행·열 의미
행렬크기행 (row) 의미열 (column) 의미원소 [i, j]
A n × n 선택된 구조 cfgi (i = 1…n) MTP 파라미터 θj (j = 1…n) ∂E(cfgi) / ∂θj
B m × n 후보 구조 newi (i = 1…m) MTP 파라미터 θj (j = 1…n) ∂E(newi) / ∂θj
B·A⁻¹ m × n 후보 구조 newi basis 방향 j (A⁻¹의 j번째 열) 후보 구조 i의 j번째 basis 방향 투영값

A와 B는 열 구조가 동일합니다 — 둘 다 n개 MTP 파라미터 방향으로 gradient를 계산합니다. 차이는 행 수: A는 정방행렬 조건으로 반드시 n행, B는 m행 (m = 임의, 1 이상). 즉 A는 "이미 선택된 n개 구조의 fingerprint 모음", B는 "새로 평가할 m개 구조의 fingerprint 모음"입니다.

4. Extrapolation Grade γ — 새 구조의 낯섦 척도

4.1 정의

새로운 구조의 feature vector $\boldsymbol{\varphi}_\text{new} \in \mathbb{R}^n$에 대해 extrapolation grade는:

$$\boxed{\gamma = \max_{j=1,\ldots,n} \left| \left(\mathbf{A}^{-1}\right)_j^\top \boldsymbol{\varphi}_\text{new} \right| = \left\| \boldsymbol{\varphi}_\text{new}^\top \mathbf{A}^{-1} \right\|_\infty}$$

여기서 $\left(\mathbf{A}^{-1}\right)_j$는 $\mathbf{A}^{-1}$의 $j$번째 열(column)입니다.

코드에서는 $m$개의 후보 구조를 한꺼번에 처리합니다:

$$\mathbf{C} = \mathbf{B} \mathbf{A}^{-\top}, \quad \gamma = \max_{i,j} |C_{ij}|$$ 
// maxvol.cpp:157–160  — BinvA = B · Aᵀ⁻¹ 계산
cblas_dgemm(CblasRowMajor, CblasNoTrans, CblasTrans,
            m, n, n, 1,
            &B(0,0), n,    // B: m×n
            &invA(0,0), n, // invA: n×n  (column-ordered, hence Trans)
            0, &BinvA(0,0), n);

swap_grade = FindMaxElem(BinvA, m); // γ = max|BinvA|

4.2 기하학적 의미

$\gamma$는 "새 구조를 basis에 추가했을 때 평행육면체 부피가 몇 배 커지는가"를 나타냅니다.

$$\frac{|\det(\mathbf{A}_\text{new})|}{|\det(\mathbf{A})|} = \gamma$$

여기서 $\mathbf{A}_\text{new}$는 $\mathbf{A}$의 한 행을 $\boldsymbol{\varphi}_\text{new}$로 교체한 행렬입니다 (best-swap 가정).

4.3 threshold 기준

조건의미MLIP 동작
$\gamma \le \gamma_\text{select}$ (예: 3.0) 기존 basis로 충분히 표현 가능 — 안전 영역 정상 진행, 저장 안 함
$\gamma_\text{select} < \gamma \le \gamma_\text{break}$ Extrapolation 발생 — 새 정보 포함 B-preselected.cfg에 저장
$\gamma > \gamma_\text{break}$ (예: 10.0) 심각한 외삽 — 포텐셜 신뢰 불가 프로그램 강제 종료

5. MaxVol Swap — 행렬 갱신 알고리즘

$\gamma > \gamma_\text{swap}$ (≈1.0000001)이면 현재 basis의 어느 행을 새 구조로 교체했을 때 det가 가장 많이 증가하는지 찾아 교체합니다.

5.1 Best-Swap 선택

$$(i^*, j^*) = \arg\max_{i,j} \left|(\mathbf{B}\mathbf{A}^{-\top})_{ij}\right|$$
  • $i^*$: 교체될 새 구조 인덱스 (B의 행)
  • $j^*$: 교체될 기존 구조 인덱스 (A의 행)

이 조합이 교체 후 $|\det(\mathbf{A})|$를 $\gamma$배 증가시킵니다.

5.2 반복 종료 조건

$$\text{while} \quad \gamma > \gamma_\text{swap} \approx 1 \quad \text{do swap}$$ 
// maxvol.cpp:182–192  — 반복 swap 루프
for (int swap_cntr = 0;
     swap_grade > threshold && swap_cntr < swap_limit;
     swap_cntr++)
{
    log_n_det += log(swap_grade)/log(n); // log|det| 누적
    Iterate(swap_grade, m);           // invA 갱신 + swap
}

수렴 시 $\mathbf{A}$의 모든 행은 해당 basis 부피를 줄이지 않는 최적 구조로 채워지며, 이것이 바로 maximum volume submatrix입니다.

6. Sherman–Morrison–Woodbury 역행렬 갱신

Swap마다 $\mathbf{A}$의 한 행만 바뀌므로, $\mathbf{A}^{-1}$을 매번 새로 계산하면 $O(n^3)$이 필요합니다. 대신 rank-1 update 공식을 씁니다.

6.1 수식 유도

A의 $j^*$번째 행 $\mathbf{a}_{j^*}$를 새 행 $\mathbf{b}_{i^*}$로 교체합니다:

$$\mathbf{A}_\text{new} = \mathbf{A} + \underbrace{(\mathbf{b}_{i^*} - \mathbf{a}_{j^*})}_{\mathbf{u}} \mathbf{e}_{j^*}^\top$$

Sherman–Morrison 공식 적용:

$$\mathbf{A}_\text{new}^{-1} = \mathbf{A}^{-1} - \frac{\mathbf{A}^{-1}\mathbf{e}_{j^*} \cdot \mathbf{u}^\top \mathbf{A}^{-1}} {1 + \mathbf{e}_{j^*}^\top \mathbf{A}^{-1} \mathbf{u}}$$

분모 $1 + \mathbf{e}_{j^*}^\top \mathbf{A}^{-1} \mathbf{u} = (\mathbf{B}\mathbf{A}^{-\top})_{i^* j^*}$ 이므로, 이미 계산된 $\gamma$값을 재활용합니다:

$$\mathbf{A}_\text{new}^{-1} = \mathbf{A}^{-1} - \frac{(\mathbf{A}^{-1})_{:,j^*} \cdot \mathbf{u}^\top \mathbf{A}^{-1}} {(\mathbf{B}\mathbf{A}^{-\top})_{i^* j^*}}$$

6.2 코드 구현 (3-step)

// maxvol.cpp:66–82  — UpdateInvA()

// Step 0: u = B[i*][:] - A[j*][:]
memcpy(&bufVect1[0], &B(i0,0), n*sizeof(double));
cblas_daxpy(n, -1, &A(j0,0), 1, &bufVect1[0], 1);

// Step 1: scalar = 1 / BinvA[i*][j*]  (분모)
double tmp = 1.0/BinvA(i0, j0);

// Step 2: v = invA의 j* 열
memcpy(&bufVect2[0], &invA(j0,0), n*sizeof(double));

// Step 3: w = tmp * u^T * invA  (분자 행벡터)
cblas_dgemv(..., tmp, &invA(0,0), n, &bufVect1[0], ...);

// Step 4: invA -= v ⊗ w  (rank-1 update)
cblas_dger(..., -1.0, &bufVect2[0], &bufVect3[0], &invA(0,0), n);

7. MLIP Active Learning 전체 흐름

MaxVol이 Step A, B, C에서 어떻게 사용되는지 통합하여 봅니다.

각 Step에서 MaxVol이 하는 일
A
Basis 구축: train.cfg 전체를 보고 MaxVol로 $\mathbf{A}$ 완성. train.cfg 중 가장 독립적인 $n$개가 $\mathbf{A}$의 행이 됨. 결과를 A-state.als로 저장.
B
Grade 평가: A-state.als 로드 후, relaxation 중 생성되는 각 구조에 대해 $\gamma = \max|\boldsymbol{\varphi}_\text{new}^\top \mathbf{A}^{-1}|$ 실시간 계산. $\gamma > \gamma_\text{select}$이면 B-preselected.cfg에 축적.
C
중복 제거 + 선택: B-preselected.cfg를 train.cfg의 basis와 비교, MaxVol로 새 정보를 가장 많이 추가하는 구조만 C-selected.cfg로 선별.
